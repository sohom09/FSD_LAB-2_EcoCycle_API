🔍 How the Notifications API Works Here

We first call Notification.requestPermission() to get user consent — it returns a Promise resolving to 'granted', 'denied', or 'default'. Once granted, we create notifications with new Notification(title, { body, icon, badge, tag }). We handle click events on notifications with notification.onclick. The Notification.permission property lets us check the current status without prompting again. Permissions are per-origin and persist across sessions.
